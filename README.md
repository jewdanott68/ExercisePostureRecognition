# ExercisePostureRecognition
An AI-powered web app that analyzes uploaded exercise videos to recognize push-up, squat, and lunge postures, count repetitions, and assess form using MediaPipe and deep learning, helping users exercise correctly and reduce injury risk.

ส่วนประกอบของโปรเจค
ระบบถูกแบ่งออกเป็น 3 ส่วนหลัก ดังนี้:
ส่วนที่ 1: การจัดเตรียมข้อมูล (Data Preparation) - 10 ไฟล์
โฟกัสที่การสกัดจุดพิกัดร่างกาย (Landmarks) จากวิดีโอและบันทึกเป็นไฟล์ข้อมูล (CSV)
  main.py
    หน้าที่: เป็นสคริปต์หลัก (Entry Point) ที่รันกระบวนการสกัดข้อมูลทั้งหมด ดึงวิดีโอมาประมวลผล และสั่งบันทึกไฟล์
  config.py
    หน้าที่: เก็บค่าการตั้งค่า (Configuration) ทั้งหมดของโปรเจค เช่น ชื่อท่าทาง, FPS ที่ต้องการ, รายชื่อจุดพิกัดที่ใช้
  dataset_loader.py
    หน้าที่: จัดการอ่านรายชื่อไฟล์วิดีโอทั้งหมดจากโฟลเดอร์ชุดข้อมูล (dataset/) แยกตามคลาสท่าทาง
  pose_estimator.py
    หน้าที่: โหลดและเรียกใช้งานโมเดล MediaPipe Pose เพื่อตรวจจับพิกัดร่างกายจากภาพ
  preprocessor.py
    หน้าที่: คัดกรองเฉพาะพิกัดที่ต้องการใช้งาน และทำ Normalization (ปรับบรรทัดฐาน) ข้อมูลพิกัดให้เป็นสัดส่วนเดียวกัน
  image_processor.py
    หน้าที่: จัดการแปลงภาพ เช่น แปลงสี BGR เป็น RGB และการพลิกภาพซ้าย-ขวา (Data Augmentation)
  fps_aligner.py
    หน้าที่: ปรับจำนวนเฟรมของวิดีโอ (Interpolation) ให้ตรงกับ FPS เป้าหมายที่ตั้งไว้ใน config.py
  frame_filler.py
    หน้าที่: เติมเต็มข้อมูลพิกัดในเฟรมที่หลุดหายไปจากการตรวจจับ โดยใช้วิธีการประมาณค่า (Interpolation) ระหว่างเฟรม
  landmark_filler.py
    หน้าที่: เติมเต็มข้อมูลเฉพาะจุดพิกัด (Landmark) บางจุดที่มองไม่เห็น (Visibility ต่ำ) ให้มีความต่อเนื่อง
csv_manager.py
    หน้าที่: จัดการสร้าง เขียน และบันทึกข้อมูลพิกัด (Features) ลงในไฟล์รูปแบบ CSV

ส่วนที่ 2: การฝึกสอนโมเดล (Model Training) - 7 ไฟล์
โฟกัสที่การนำไฟล์ CSV มาเตรียมข้อมูลและสอนโมเดล Deep Learning
  dataset_reader.py
    หน้าที่: อ่านข้อมูลจากโฟลเดอร์ CSV, จัดการแบ่งข้อมูล (Train/Val/Test) และสร้าง Sliding Windows เพื่อป้อนเข้าโมเดล
  model_utils.py
    หน้าที่: รวมฟังก์ชันช่วยเหลือในการเทรน เช่น การบันทึก Log, วาดกราฟความแม่นยำ (Accuracy), และสร้าง Confusion Matrix
  train_cnn_1layer.py
    หน้าที่: สคริปต์สำหรับสร้างและฝึกสอนโมเดลโครงข่ายประสาทเทียมแบบคอนโวลูชัน 1 มิติ (1D-CNN) ขนาด 1 เลเยอร์
  train_cnn_2layer.py
    หน้าที่: สคริปต์สำหรับสร้างและฝึกสอนโมเดล 1D-CNN ขนาด 2 เลเยอร์ (มีความซับซ้อนขึ้น)
  train_lstm_1layer.py
    หน้าที่: สคริปต์สำหรับสร้างและฝึกสอนโมเดลหน่วยความจำระยะสั้นแบบยาว (LSTM) ขนาด 1 เลเยอร์
  train_lstm_2layer.py
    หน้าที่: สคริปต์สำหรับสร้างและฝึกสอนโมเดล LSTM ขนาด 2 เลเยอร์
  train_hybrid.py
    หน้าที่: สคริปต์สำหรับสร้างและฝึกสอนโมเดลแบบผสมผสาน (Hybrid) ที่ใช้ทั้ง 1D-CNN และ LSTM ร่วมกัน

ส่วนที่ 3: เว็บแอปพลิเคชัน (Web App) - 2 ไฟล์
โฟกัสที่การนำโมเดลไปใช้งานจริงผ่านระบบ Interface และการวิเคราะห์ผลลัพธ์
  app.py
    หน้าที่: เป็นไฟล์หลักของ Streamlit สร้างหน้าเว็บรับอัปโหลดวิดีโอ, เรียกใช้โมเดลทำนายผล, วาดกราฟิกบนภาพ, และสรุปผล
  rep_counter.py
    หน้าที่: คำนวณมุมองศาข้อต่อของร่างกาย ตรวจสอบเกณฑ์การนับจำนวนครั้ง (Repetitions) และสร้างข้อความคำแนะนำท่าทาง (Feedback)

ขั้นตอนการติดตั้ง
ติดตั้งสภาพแวดล้อม Python (แนะนำเวอร์ชัน 3.8 หรือ 3.9)
  ติดตั้งไลบรารีที่จำเป็นผ่านคำสั่ง pip: pip install opencv-python mediapipe tensorflow streamlit pandas numpy scikit-learn matplotlib altair

ขั้นตอนการใช้งาน
1. การเตรียมชุดข้อมูลเพื่อฝึกสอน
  จัดเก็บวิดีโอลงในโฟลเดอร์ dataset/ โดยแยกโฟลเดอร์ย่อยตามชื่อท่าทาง
  รันคำสั่งเพื่อสกัดข้อมูลเป็น CSV: python main.py
2. การฝึกสอนโมเดลใหม่
  เลือกสถาปัตยกรรมที่ต้องการและรันคำสั่งเทรน ตัวอย่างเช่น: python train_cnn_1layer.py
  โมเดลที่ผ่านการฝึกสอนแล้วจะถูกบันทึกไว้ในโฟลเดอร์ models/ ในรูปแบบไฟล์ .keras
3. การรันระบบแอปพลิเคชัน
  ตรวจสอบให้มั่นใจว่ามีไฟล์โมเดลอยู่ในโฟลเดอร์ models/ จากนั้นใช้คำสั่ง: streamlit run app.py

รายละเอียดการตั้งค่า (Configuration)
ผู้พัฒนาสามารถปรับปรุงค่าคงที่ต่างๆ ได้ในไฟล์ config.py เช่น:
  TARGET_FPS: อัตราเฟรมเรตที่ต้องการประมวลผล
  SEQUENCE_LENGTH: จำนวนเฟรมต่อเนื่องที่ใช้ในการทำนายผล
  TARGET_LANDMARKS: จุดพิกัดร่างกายที่ต้องการใช้งาน
